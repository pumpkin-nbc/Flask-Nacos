# 1.0.0 发布前验收清单

[English](1.0-checklist.md) | 简体中文

`0.9.0` 是 `1.0.0` 之前的 Release Candidate 准备版本。从 `0.9.0` 起，公开 API 被视为
冻结的 `1.0.0` 候选。本清单汇总了发布 `1.0.0` 之前必须验证的所有事项。

另请参阅：[发布指南](release.zh-CN.md) - [兼容性](compatibility.zh-CN.md) -
[API 参考](api-reference.zh-CN.md)。

## 验收清单

1. API 已冻结（`scripts/check_api_snapshot.py` 通过）。
2. `README.md` 完整且为最新。
3. `docs/` 完整（每篇文档都有中英文两个版本）。
4. `examples/` 完整且通过 `scripts/check_examples.py`。
5. `CHANGELOG.md` 完整，且最新标题与版本号一致。
6. `pyproject.toml` 元数据完整（名称、版本、描述、作者、许可证、分类器、URL、
   Python 版本要求）。
7. 已完成 TestPyPI 上传与安装验证。
8. 安装包 smoke test 通过（`scripts/smoke_test_package.py`）。
9. 已在 Python 3.8 - 3.12 验证通过。
10. 已在 Flask `>=1.0,<4.0` 验证通过。
11. 不包含任何敏感信息（`scripts/check_sensitive_info.py`）。
12. 不包含内部 IP 或内部域名。
13. 不提供 `get_config_as_dict`。
14. 不提供 `load_config_to_flask`。
15. 未实现 YAML、JSON、dict 配置解析。
16. `get_config()` 只返回配置原始内容（不做解析）。
17. 所有测试通过（`pytest`）。
18. 构建成功（`python -m build`）。
19. `twine check dist/*` 通过。
20. 已验证安装已发布的包并成功 import。

## 如何执行完整检查

```bash
bash scripts/release_check.sh
```

它会运行上述所有检查（不含真实的 TestPyPI/PyPI 上传，上传是
[发布指南](release.zh-CN.md)中描述的独立且需显式执行的步骤）。
