# 1.0.0 Release Checklist

English | [简体中文](1.0-checklist.zh-CN.md)

`0.9.0` is the Release Candidate preparation version before `1.0.0`. From
`0.9.0` the public API is treated as the frozen `1.0.0` candidate. This
checklist gathers everything that must be verified before cutting `1.0.0`.

See also: [Release Guide](release.md) - [Compatibility](compatibility.md) -
[API Reference](api-reference.md).

## Acceptance checklist

1. API is frozen (`scripts/check_api_snapshot.py` passes).
2. `README.md` is complete and up to date.
3. `docs/` is complete (every page has an English and 简体中文 version).
4. `examples/` is complete and passes `scripts/check_examples.py`.
5. `CHANGELOG.md` is complete and its newest heading matches the version.
6. `pyproject.toml` metadata is complete (name, version, description, authors,
   license, classifiers, URLs, Python requirement).
7. TestPyPI upload and install has been validated.
8. Package smoke test passes (`scripts/smoke_test_package.py`).
9. Python 3.8 - 3.12 has been validated.
10. Flask `>=1.0,<4.0` has been validated.
11. No sensitive information is present (`scripts/check_sensitive_info.py`).
12. No internal IPs or internal domains are present.
13. `get_config_as_dict` is not provided.
14. `load_config_to_flask` is not provided.
15. No YAML, JSON, or dict config parsing is implemented.
16. `get_config()` returns raw config content only (no parsing).
17. All tests pass (`pytest`).
18. Build succeeds (`python -m build`).
19. `twine check dist/*` passes.
20. Installing the published package and importing it has been verified.

## How to run the full check

```bash
bash scripts/release_check.sh
```

This runs every check above (except the live TestPyPI/PyPI upload, which is a
separate, explicit step described in the [Release Guide](release.md)).
