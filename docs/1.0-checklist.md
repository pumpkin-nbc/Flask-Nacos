# 1.0.0 Release Checklist

English | [简体中文](1.0-checklist.zh-CN.md)

`1.0.0` is the first stable release of Flask-Nacos, intended for PyPI. The
public API is stable for the 1.0 series. This checklist gathers everything that
must be verified before cutting `1.0.0`.

See also: [Release Guide](release.md) - [Compatibility](compatibility.md) -
[API Reference](api-reference.md).

## Acceptance checklist

1. API is frozen (`scripts/check_api_snapshot.py` passes).
2. Version is updated to `1.0.0` (pyproject and `__version__`).
3. `CHANGELOG.md` has the `1.0.0` entry.
4. `README.md` has been finalized.
5. `docs/` has been finalized (every page has an English and 简体中文 version).
6. `examples/` have been finalized.
7. `pyproject.toml` metadata has been finalized (name, version, readme,
   requires-python, dependencies, dev extras, classifiers, license, URLs).
8. `py.typed` is included in the package.
9. No sensitive information is present.
10. No internal IPs are present.
11. No internal domains are present.
12. `get_config_as_dict` is not provided.
13. `load_config_to_flask` is not provided.
14. No YAML, JSON, or dict config parsing is implemented.
15. `get_config()` returns raw config content only (no parsing).
16. `pytest` passes.
17. `ruff check .` passes.
18. `mypy flask_nacos` passes.
19. `scripts/check_version.py` passes.
20. `scripts/check_sensitive_info.py` passes.
21. `scripts/check_docs.py` passes.
22. `scripts/check_compatibility.py` passes.
23. `scripts/check_api_snapshot.py` passes.
24. `scripts/check_examples.py` passes.
25. `python -m build` passes.
26. `twine check dist/*` passes.
27. `scripts/check_package.py` passes.
28. `scripts/smoke_test_package.py` passes.
29. TestPyPI publish and install has been validated.
30. PyPI publish is prepared.
31. The `v1.0.0` Git tag is prepared.
32. The GitHub Release notes are prepared.

## How to run the full check

```bash
bash scripts/release_check.sh
```

This runs every check above (except the live TestPyPI/PyPI upload, which is a
separate, explicit step described in the [Release Guide](release.md)).
