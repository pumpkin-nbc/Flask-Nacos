# 1.0.0 Release Checklist

English | [简体中文](1.0-checklist.zh-CN.md)

`1.0.0` is the first stable release of Flask-Nacos, intended for PyPI. The
public API is stable for the 1.0 series. This checklist gathers everything that
must be verified before cutting `1.0.0`.

See also: [Release Guide](release.md) - [Compatibility](compatibility.md) -
[API Reference](api-reference.md).

## Acceptance checklist

1. API is frozen (`scripts/check_api_snapshot.py` passes).
2. Version is updated to `1.0.0` (pyproject and `__version__`).
3. `CHANGELOG.md` has the `1.0.0` entry.
4. `README.md` has been finalized.
5. `docs/` has been finalized (every page has an English and 简体中文 version).
6. `examples/` have been finalized.
7. `pyproject.toml` metadata has been finalized (name, version, readme,
   requires-python, dependencies, dev extras, classifiers, license, URLs).
8. `README.zh-CN.md`, `SECURITY.md`, `LICENSE`, and `NOTICE` are included in the sdist.
9. `py.typed` is included in the package.
10. No sensitive information is present.
11. No internal IPs are present.
12. No internal domains are present.
13. `get_config_as_dict` is not provided.
14. `load_config_to_flask` is not provided.
15. No YAML, JSON, or dict config parsing is implemented.
16. `get_config()` returns raw config content only (no parsing).
17. The authenticated config round-trip integration test passes on non-production Nacos.
18. The ephemeral heartbeat integration test passes beyond the deletion window.
19. `pytest` passes.
20. `ruff check .` passes.
21. `mypy flask_nacos` passes.
22. All consistency scripts pass.
23. `python -m build` passes from a clean checkout.
24. `twine check --strict dist/*` passes.
25. `scripts/check_package.py` confirms metadata, contents, and source freshness.
26. `scripts/smoke_test_package.py` installs and checks both wheel and sdist.
27. The release commit is merged to `master` and its full CI matrix is green.
28. The `testpypi` and protected `pypi` GitHub Environments are configured.
29. Matching Pending Trusted Publishers are configured on both indexes.
30. TestPyPI publishing and Python 3.8/3.13 installation are validated.
31. The protected `v1.0.0` Git tag is prepared.
32. The GitHub Release notes are prepared.

## How to run the full check

```bash
bash scripts/release_check.sh
```

This runs every check above (except the live TestPyPI/PyPI upload, which is a
separate, explicit step described in the [Release Guide](release.md)).
